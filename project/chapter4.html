4.1 Software Types

System software, application software and utility programs.

The Software Hierarchy

APPLICATION SOFTWARE — what the user runs
↓
SYSTEM SOFTWARE — makes hardware usable
↓
HARDWARE — CPU, RAM, storage…

Every computer begins life as a pile of connected hardware — a CPU, some memory, a few peripherals. Without software, none of it moves. The software stack solves that in two layers: system software that wakes the hardware up and keeps it sane, and application software that lets a person actually do something useful with the machine.

General features of system software

  • A set of programs that control and manage the operation of computer hardware
  • Provides a platform on which other software can run
  • Lets hardware and software run together without conflicts
  • Provides the Human Computer Interface (HCI)
  • Controls the allocation and usage of hardware resources

General features of application software

  • Used to perform specific applications (apps) on a computer
  • Lets the user perform specific tasks using the computer's resources
  • May be a single program (Notepad) or a suite (Microsoft Office)
  • User executes it as and when they require

Utility programs are a sub-category of system software — small housekeeping tools (anti-virus, backup, defragmentation, file compression). Some run quietly in the background (a virus checker), others the user starts deliberately (file compression). A linker (also system software) takes the object files produced by a compiler from different modules and combines them into one runnable program.

Application Software — Types & Examples

Application software is what most people picture when they hear the word "computer". It lets a user perform specific tasks using the machine's resources — write an essay, balance accounts, edit a holiday photo, model a chemical plant. It can be a single program (Notepad) or a suite of related programs sold together (Microsoft Office, Adobe Creative Cloud). The user launches it on demand; it sits on top of the operating system and uses the OS to reach the hardware.

Spreadsheet

Organises and manipulates numerical data (integers, real numbers, dates) on a grid of lettered columns and numbered rows. Each cell has a unique address such as B6. Key features: formulas that recalculate automatically, the ability to produce graphs from data, and modelling / "what-if" calculations — change one input and watch every dependent cell update.

Database

Organises, manipulates and analyses data stored in one or more tables. Each row is a record (one customer, one product) and each column is a field (Name, Price, Quantity). Supports queries to pull out matching records, reports that format the results, and add / delete / modify operations on individual records.

Word Processor

Manipulates text documents such as essays or reports. Text is entered using a keyboard and the software supplies the tools: create, edit, save; copy & paste; spell-checker and thesaurus; importing photos into a structured page layout; even translation into a foreign language.

Apps (Mobile)

"Apps" is just short for applications — the term has stuck for software that runs on phones and tablets. Downloaded from an App Store, ranging from games to sophisticated banking. Common examples: video / music streaming, GPS navigation, and camera apps that capture, store and manipulate photos.

Control & Measuring

Interfaces a computer or microprocessor with sensors. Two main roles: measure physical quantities in the real world (e.g. temperature), and control applications by comparing sensor data with stored values and sending signals back — for example, opening a valve to add acid to a chemical mix and shift the pH.

Video Editing

Manipulates raw video to produce a finished film. Allows the user to rearrange, add and remove sections of video or audio, apply colour correction, filters and other enhancements, add titles, alter the soundtrack, and create smooth transitions between clips.

Graphics Manipulation

Edits images in two very different ways. Bitmap editors change individual pixels (each carrying brightness and colour). Vector editors manipulate lines, curves and text rather than pixels — so the result scales smoothly to any size. The choice depends on the original image's format.

Photo Editing

Lets a user manipulate digital photographs — change brightness, contrast, colour saturation, or remove red-eye. Also supports very complex effects: changing facial features, combining photos, removing unwanted items, and "touching up" a shot to make it as perfect as possible.

System Software — Utility Programs

Utilities are part of system software designed to help manage, maintain and control computer resources. Some run in the background without any user input — a virus checker is the obvious example — while others wait for the user to start them, such as a file-compression tool you reach for when emailing a folder of photos. They are small, focused tools that keep the system healthy and the data safe.

UtilityWhat it does
Virus checker (anti-virus) Runs in the background scanning every file before it loads. Compares files against a database of known viruses; performs heuristic checking — looking for behaviour that suggests a virus, even one not yet in the database. Suspect files are placed in quarantine for automatic deletion, or for the user to review. A clean file flagged as infected is called a false positive — the main drawback of heuristic detection. Must be kept up to date as new viruses appear daily; a full system scan is recommended weekly.
Defragmentation Over time, as files are deleted, extended and rewritten, an HDD's data ends up scattered across non-adjacent sectors and tracks — every read needs many head movements. A defragmenter rearranges the blocks so each file occupies contiguous sectors, slashing access time. Not needed for SSDs, which have no moving heads and read any block in equal time.
Back-up software Schedules automatic copies and only backs up files that have changed since last time (saving space). Best practice: three versions of every file — (1) the working copy on the internal HDD/SSD, (2) a local backup on a portable drive, (3) a remote backup well away from the computer (e.g. cloud storage). Windows uses File History (snapshots every hour to an external HDD, kept forever by default); Mac OS uses Time Machine (hourly for a day, daily for a month, weekly thereafter — oldest backups deleted when the drive fills).
Security software An umbrella utility. Manages access control and user accounts; integrates with anti-virus and anti-spyware; protects network interfaces (firewalls); uses encryption / decryption so intercepted data is meaningless without a key; oversees software updates — verifying each update comes from a legitimate source.
Screensaver Originally invented to protect old CRT monitors from "phosphor burn" caused by a static image — irrelevant for modern LCD / OLED panels. Today screensavers play three roles: (1) personalising the desktop, (2) acting as a security lock — after a few minutes of inactivity the user is automatically logged out, (3) triggering background tasks that should only run when the machine is idle, such as a virus scan or a distributed-computing job.
Device drivers Software that talks to the operating system and translates data into a format that a specific hardware peripheral understands. The instant a USB device is plugged in, the OS hunts for the matching driver — fail to find one and you see "device not recognised". Every USB driver contains descriptors: Vendor ID (VID), Product ID (PID) and a unique serial number. Without a serial number, the OS treats the device as new every single time it's plugged in.
Defragmentation illustrated
When files are deleted and extended over time, data blocks scatter across different sectors. Example — File 1 ends up in sectors 0–3 and 10–11, File 3 spans two tracks. The defragmenter rewrites blocks so each file occupies consecutive sectors on one track, cutting head movements dramatically.

4.2 Operating Systems

The 9 functions of an OS, plus CLI vs GUI.

9 Functions of an OS

An operating system is software running quietly in the background of every computer. Without it, most machines would be almost unusable — you'd be talking to bare hardware. It does three jobs at once: it provides an environment in which applications can run, it gives humans an interface to drive the machine, and it disguises the complexity of software and hardware underneath. Familiar examples include Microsoft Windows, Apple macOS, Linux, Google Android and Apple iOS (the last two designed primarily for phones and tablets). On a desktop the OS lives on the HDD or SSD because it is large; on a phone it sits on solid-state storage for the same reason — there's simply no room for a hard disk.

1. HCI

Provides CLI or GUI for user interaction.

2. Multitasking

Runs several programs by sharing CPU time.

3. Memory management

Allocates RAM to programs; prevents interference.

4. Security

Manages logins, passwords and permissions.

5. Peripheral management

Controls I/O devices via device drivers.

6. File management

Create, open, save, copy, delete and organise files.

7. Interrupt handling

Responds to signals from devices or programs.

8. Platform for apps

Provides the environment applications need to run.

9. User accounts

Who uses each? CLI: programmers, analysts, technicians — anyone who needs direct communication with the computer to develop software, find errors and remove them, or initiate memory dumps (capture the exact contents of memory at one instant). GUI: end-users who don't have, and don't need, deep knowledge of how the computer works — running apps, playing games, browsing photos.

WIMP & post-WIMP. The standard PC paradigm is WIMP — Windows, Icons, Menus, Pointing device — built around the mouse. Each window holds an application; modern systems let several be open at once. A windows manager orchestrates the interaction between windows, applications, the pointer and the cursor's position. Phones and tablets use post-WIMP: fingers directly on the screen, with gestures like pinching and rotating that would be awkward with a single mouse pointer.

OS Functions — In Depth

Memory Management

Controls primary storage (RAM) while programs are running. The OS:

  • Moves data between RAM and HDD/SSD during program execution — keeping the most-used pages in fast memory
  • Keeps track of every memory location: what's free, what's allocated, and to which process
  • Carries out memory protection — stops two competing applications using the same memory location at the same time. Without it: data could be lost; applications could produce incorrect results from corrupted values; misplaced data could create a security hole by becoming visible to other software; in extreme cases the whole computer could crash.

Security Management

Ensures the integrity, confidentiality and availability of data by:

  • Applying OS updates and keeping anti-virus software current
  • Communicating with firewalls to check all incoming/outgoing traffic
  • Setting up user accounts, passwords and user IDs with privilege levels
  • Maintaining access rights and offering data recovery / system restore

Hardware Peripheral Management

Controls all I/O devices using device drivers. Example — printing a document:

  1. Printer driver is located and loaded into memory
  2. Data is sent to the printer buffer
  3. If the printer is busy or low priority, data queues in the print queue
  4. Control commands are sent throughout the printing process
  5. Error messages and interrupts from the printer are received and handled

File Management

  • Enforces naming conventions (e.g. report.docx, data.csv)
  • Performs operations: create, open, close, delete, rename, copy, move
  • Maintains directory structures and access rights
  • Allocates memory — reads files from HDD/SSD into RAM

Multitasking

Lets a computer carry out more than one process at a time. Each process shares the hardware resources under OS control. To stop processes clashing, the OS constantly monitors the status of each one. The three rules below are collectively known as pre-emptive multitasking:

  • Resources are allocated to a process for a specific time limit
  • The process can be interrupted while it is running
  • Each process is given a priority — the risk is that a very low-priority process can be starved of resources

Through good multitasking management, main memory, HDD/SSD and virtual memory are all used more efficiently — squeezing the most out of CPU time.

User Accounts

Each user logs in with a username and password. The OS:

  • Stores each user's files in separate, protected memory areas
  • Lets users customise their own screen layout and settings
  • Allows an administrator to create, delete and restrict accounts
  • Enforces multi-access levels — e.g. a clerk can order stationery but cannot access personal data

BIOS, EEPROM & CMOS

When you press the power button, the OS isn't there yet — it's still on the hard drive. Something has to be running already to fetch it. That something is the BIOS (Basic Input/Output System) — a small program permanently stored on the motherboard. The BIOS tells the computer where the OS lives, loads the bit of the OS that's needed, and hands control over. Because it provides low-level control for hardware, the BIOS is called firmware.

ComponentWhat it storesMemory typeKey property
BIOS software The actual BIOS program — POST routines, bootstrap loader EEPROM (flash ROM chip) Non-volatile; can be rewritten/updated by user
BIOS settings User-configured values: clock speed, boot order, date/time CMOS chip (battery-backed) Loses settings if motherboard battery is removed
OS & apps Operating system + application software HDD or SSD Loaded into RAM when needed
Running program Part of OS or app currently executing RAM Volatile — lost on power-off
What happens at power-on?
1 → CPU starts; BIOS runs from EEPROM.
2 → BIOS runs POST (Power-On Self Test) — checks CPU, RAM and keyboard.
3 → BIOS reads CMOS for boot-device settings, locates the OS on that device.
4 → Bootstrap loader copies the OS kernel into RAM.
5 → OS takes over; loads device drivers and the user interface.
6 → Application software can now be launched by the user.

Firmware = a program providing low-level control for a device, stored on ROM / EEPROM. The BIOS is the best-known example. EEPROM (Electrically Erasable Programmable ROM) is a flash memory chip — its contents survive power-off, yet it can still be rewritten, updated, or even deleted by a user. The BIOS program lives there. The BIOS settings — clock speed, boot order, date and time — live on the separate CMOS chip, which is kept alive by a small rechargeable battery on the motherboard. Pull that battery and the CMOS forgets everything: when power returns, the BIOS will reload but every setting reverts to factory defaults. So if a user tweaks the clock speed and then the battery dies, their tweak is gone.

Once the OS is up and running, the application software runs on top of it, accessing system software (such as device drivers) as needed. Different parts of the OS itself get loaded in and out of RAM as the software runs — there's never a single "the OS" sitting still in memory.

4.4 Interrupts

How devices and software get the CPU's attention.

What is an Interrupt?

An interrupt is a signal sent from a device or from software to the microprocessor. The signal causes the CPU to temporarily stop what it's doing so it can service the interrupt — run a short piece of code called the Interrupt Service Routine (ISR). When the ISR finishes, the saved state is restored and the original task carries on as though nothing happened. Interrupts are the reason you can download a file while listening to music: data is passing in and out of memory so rapidly that both tasks appear to be running simultaneously.

5 Causes

  • Timing signal — the system clock or a timer needs servicing on a schedule
  • I/O process — a printer or disk drive needs more data, or has finished a job
  • Hardware fault — a paper jam in the printer, low battery, a disk error
  • User interaction — keys like <CTRL><ALT><BREAK>, a mouse click, or any key-press
  • Software error — an .exe file that cannot be found, two processes accessing the same memory location, or an attempt to divide by zero

Once the signal arrives, the CPU either carries on or stops to service it, depending on the interrupt's type and priority. A high-priority interrupt (e.g. a thermal warning) is dealt with immediately; a low-priority one (timer tick) may be queued.

Buffers & Why They Matter

A buffer is a temporary memory area used to compensate for the speed difference between a fast device (CPU) and a slow device (printer, disk, network).

Printing example

Document data is sent from memory to the printer buffer. The CPU can refill the buffer in microseconds; the printer drains it over seconds. Without a buffer the CPU would be locked waiting for each page to print — wasting processing time.

Video streaming example

Data downloads from the web server into a stream buffer. The buffer absorbs network speed fluctuations. If the download rate drops below the playback rate, the buffer empties and the video "freezes".

How interrupts + buffers work together (printing)

  1. Document data is sent from memory to the printer buffer.
  2. The CPU continues its own tasks (it does not wait idle).
  3. When the printer buffer empties, the printer sends an interrupt signal to the CPU.
  4. The CPU saves its current state (PC + registers → stack).
  5. The ISR runs — checks if more data exists and refills the buffer.
  6. If all data has been printed, no more data is sent; interrupt is cleared.
  7. The CPU restores its saved state and resumes its original task.
Key insight: Buffers and interrupts together maximise CPU utilisation. Without them, the CPU would remain idle waiting for slow I/O — an enormous waste of processing power.

The Interrupt Service Routine (ISR)

Every interrupt goes through the same sequence. Interrupts are assigned a priority so the CPU knows which to service first.

01 Interrupt signal received by the CPU from a device or software.
02 CPU checks the interrupt register — identifies the interrupt type and priority.
03 State saved — contents of the Program Counter (PC) and all registers pushed to the stack.
04 ISR executes — start address of the ISR loaded into the PC; interrupt is serviced.
05 State restored — saved registers popped from stack; interrupt register reset.
06 Original task resumes from exactly where it was paused (PC restored).

Five interrupt causes: timing signal · I/O process · hardware fault · user interaction · software error (e.g. divide by zero, missing .exe, two processes accessing the same memory location).

Extension — Interrupts in the Fetch–Execute Cycle

Beyond the syllabus, but worth peeking at if you continue to A Level. A special interrupt register sits inside the CPU. While the CPU is mid-cycle, an event (say a write failure on the hard drive) flips a bit in that register — for example, the contents change from 0000 0000 to 0000 1000. Here's what happens next:

  • At the next Fetch–Execute cycle, the interrupt register is checked bit by bit.
  • A non-zero bit shows an interrupt has occurred and still needs servicing — the CPU either services it now or, depending on priority, leaves it for a moment.
  • To service it, the CPU stops the current task and stores the contents of all its registers.
  • Control transfers to the interrupt handler / ISR.
  • Once the interrupt is fully serviced, the register is reset and the saved registers are restored.
FETCH → fetch next instruction  ·  interrupts disabled
DECODE / EXECUTE → decode and execute the instruction
INTERRUPT STAGE → interrupts enabled, priority checked → any interrupts?
    YES → suspend current program → execute ISR → resume
    NO  → continue to next cycle (unless end of program)

4.5 Programming Languages

High-level vs low-level — readability, portability and control.

High-Level vs Low-Level

High-level languages (Python, Java, C++) are close to English and portable. Low-level languages — machine code (binary) and assembly (mnemonics like LDA, ADD, STO) — are tied to a specific CPU but very fast and compact.

HLL Pros

  • Easy to read & write
  • Fast development
  • Portable across CPUs

LLL Pros

  • Direct hardware control
  • Very fast at runtime
  • Tiny memory footprint

  • Programs may run more slowly
  • Cannot always exploit special hardware
Low-level (LLL)

  • Code takes up very little primary memory
  • Tasks execute very quickly
  • Longer to write and debug
  • More difficult to understand
  • Programs are hardware-specific (not portable)

Assembly Language & Machine Code

Assembly language uses short mnemonics — one per machine instruction — instead of raw binary. An assembler translates it 1:1 into machine code.

High-level (Python)

Sum = First + Second

One readable statement. Compiles into several machine instructions.

Assembly (mnemonics)

LDA  First
ADD  Second
STO  Sum

Three mnemonics → three machine instructions (1:1 mapping).

Mnemonic meanings

MnemonicMeaning
LDA FirstLoad the value of variable First into the accumulator register
ADD SecondAdd the value of variable Second to the accumulator
STO SumStore the accumulator's value into variable Sum
Why use assembly? To use special hardware features, write machine-dependent instructions, minimise memory footprint, or maximise execution speed — e.g. in embedded systems, device drivers, or game engines.

Machine code (binary / hex)

The same three-instruction program in machine code looks like this:

HexBinary
120001 0010
4D0100 1101
1A0001 1010

Machine code is directly executable by the CPU — no translation needed. Programmers never write it by hand; an assembler or compiler produces it automatically.

4.6 Translators

Compilers, interpreters and assemblers.

Compiler vs Interpreter

A compiler translates the entire program at once into a stand-alone executable. An interpreter translates and executes one statement at a time — no executable is produced. An assembler translates assembly mnemonics into machine code (1:1 per instruction).

Code Editor

Built-in text editor — write and edit source code without switching to a separate app. Speeds up the development cycle.

Translator

Built-in compiler and/or interpreter. Typically the interpreter is used during development; the compiler produces the final distributable version.

Debugger

Runs the program under development. Allows single stepping (one line at a time) and breakpoints (pause at a chosen line). A report window shows the contents of all variables at that point, helping locate logic errors.

Error Diagnostics

Dynamic error checking — finds possible errors as code is typed and underlines them (usually in red). Alerts the programmer immediately without needing to run the code.

Auto-correction

Provides suggested corrections for flagged errors. Many simple typos (e.g. pritn → print) are corrected automatically during writing and editing.

Auto-completion

Context-sensitive prompts — suggests variable names and reserved words as you type (e.g. type pri → IDE offers print). Reduces typing and prevents name-spelling bugs.

Prettyprinting

Colour-codes keywords, strings and comments; auto-indents blocks; lays out code in a standardised readable format. Makes it easier to see structure at a glance.

Auto-documenter

Generates explanatory comments describing the function and purpose of procedures and variables — either automatically or from structured comment templates in the source code.

Software Types

Security managementThe part of an OS ensuring the integrity, confidentiality and availability of data via accounts, firewalls and encryption.
Hardware managementThe part of an OS that controls all I/O devices using device drivers; manages queues and buffers.
File managementOS function for creating, opening, closing, renaming, copying, moving and deleting files; maintains directory structures.
MultitaskingA function allowing a computer to process more than one task at a time by sharing CPU time between processes.
User accountAn agreement allowing an individual to use a computer system; protected by a username and password.
AdministratorA person responsible for managing a multi-user computer system; can create, delete and restrict user accounts.
Error handling routineA routine that recognises and recovers from abnormal inputs or hardware faults, e.g. division by zero.

Booting & Firmware

FirmwareA program providing low-level control for a device, stored on ROM or EEPROM (e.g. BIOS).
Boot up / Bootstrap loaderA small program that loads the OS into RAM to correctly start up a computer system.
BIOSBasic Input/Output System — firmware stored on EEPROM that runs POST and locates/loads the OS on power-up.
EEPROMElectrically Erasable Programmable ROM — flash memory chip storing the BIOS program; non-volatile and rewritable.
CMOSBattery-backed chip on the motherboard that stores BIOS settings (clock speed, boot order, date/time). Settings reset if battery is removed.

Interrupts & Buffers

InterruptA signal sent from a device or software to the microprocessor requesting its attention; the CPU suspends operations until the interrupt is serviced.
Interrupt priorityA ranking assigned to interrupts so the CPU knows which to service first and how urgently.
ISR (Interrupt Service Routine)Software that handles a specific interrupt request and sends the result to the CPU for processing.
BufferA temporary memory area used to hold data while a slow output device (printer, disk) processes it, freeing the CPU for other tasks.

Languages & Translators

Machine codeA binary programming language directly executed by the CPU — no translation required.
High-level language (HLL)A portable, hardware-independent language (Python, Java, C++) that must be translated into machine code before execution.
Low-level language (LLL)A hardware-dependent language — both machine code and assembly are LLLs.
Assembly languageA hardware-specific language using mnemonics (LDA, ADD, STO); translated into machine code by an assembler.
MnemonicA short code in assembly language representing one machine instruction (e.g. LDA = Load into Accumulator).
CompilerTranslates an entire HLL program into a stand-alone machine code executable in one go.
InterpreterTranslates and executes a HLL program one statement at a time; no executable is produced.
AssemblerTranslates an assembly language program into machine code (one-to-one mapping per instruction).
TranslatorGeneral term for any program (compiler, interpreter or assembler) that converts source code into machine code.

IDE

IDE (Integrated Development Environment)A suite of programs used to write, run and test source code — bundles code editor, translator, debugger and diagnostic tools.
DebuggingFinding and fixing errors in a program by running or tracing it.
PrettyprintingDisplaying source code with different colours and formatting to make it easier to read and understand.
Report windowA separate window in the IDE runtime environment showing the current contents of variables during program execution.
BreakpointA marker set in source code that pauses execution at that line so the programmer can inspect variables and program state.
Single steppingRunning a program one line at a time in the debugger to trace logic and find errors.
