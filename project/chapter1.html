1.1 Number Systems

Why computers use binary, and how binary, denary and hexadecimal relate.

Why Binary?

Computers contain millions of tiny switches — each either ON or OFF. This maps perfectly onto the binary number system, which uses only two values: 1 (ON) and 0 (OFF). Every piece of data must be converted to binary before a computer can process it.

—

No matter how complex a system seems, the basic building block is the binary digit: a bit.

Why hex is used

One hex digit replaces four binary digits — far easier for humans to read, write and remember. Common uses: error codes, MAC addresses, IPv6, HTML colour codes.

Worked Example: 00100111 + 01001010

00100111 ( 39) + 01001010 ( 74) ───────── 111 ← carries 01110001 = 113 ✓

Overflow Errors

An overflow error occurs when a result is too large for the register — a 9th bit appears in an 8-bit register.

01101110 (110) + 11011110 (222) ───────── 101001100 ← 9 bits → overflow!
—

Max value in 8-bit register = 2⁸ − 1 = 255. 16-bit = 65 535. 32-bit ≈ 4.29 billion.

Logical Shifts

A logical shift moves all bits left or right. Empty positions fill with 0.

  • Each left shift = multiply by 2
  • Each right shift = divide by 2
—

MSB = most significant bit (leftmost). LSB = least significant bit (rightmost). Bits shifted off either end are lost — losing a 1-bit corrupts the result.

1.2 Text & Character Sets

How computers encode text — ASCII and Unicode.

ASCII

ASCII — American Standard Code for Information Interchange (1963). Standard ASCII: 7-bit → 128 characters. Extended: 8-bit → 256 characters.

—

Compare 'A' (01000001) and 'a' (01100001) — only bit 6 differs. Upper↔lower case conversion flips a single bit.

Unicode

Created 1991 to represent all world languages and symbols. Uses up to 4 bytes per character. First 128 codes are identical to ASCII — fully backwards-compatible.

ASCII

  • 1 byte per character
  • 128 / 256 characters
  • English-focused
  • Smaller files

Unicode

  • Up to 4 bytes
  • 100,000+ characters
  • All world languages + emoji
  • Larger files

Sound Representation

How analogue sound waves are converted to digital binary.

Sampling

Sound is analogue — continuous. An ADC (Analogue-to-Digital Converter) measures the amplitude at regular intervals and encodes each value as binary.

Sampling Rate

Samples per second, measured in Hz. CDs use 44 100 Hz (44.1 kHz).

Sampling Resolution (Bit Depth)

Bits per sample. CDs use 16 bits → 65 536 possible amplitude levels.

Higher resolution — benefits

  • Larger dynamic range
  • Better quality
  • Less distortion

Higher resolution — costs

  • Larger file size
  • Slower transmission
  • More processing power
—

Don't confuse sampling rate (how often you measure) with sampling resolution (how precise each measurement is). Both affect quality and file size.

Image Representation

Bitmap images, pixels, colour depth and resolution.

Bitmap Images

A bitmap is a grid of pixels. Each pixel stores a binary number representing its colour.

Colour Depth

Bits/pixelColours possibleNotes
12Black and white
8256Indexed colour

File Size

File size (bits) = width × height × colour depth

Resolution

More pixels → sharper image → larger file. Fewer pixels → pixelated when enlarged → smaller file.

—

Key trade-off: a small drop in resolution often goes unnoticed by the viewer but saves significant storage space.

1.4 Compression

Reducing file sizes — lossy and lossless methods.

Why Compress?

Smaller files transmit faster, use less storage and stream more smoothly. Two types:

Lossy

  • Permanently removes data
  • Cannot be reversed
  • Smaller file
  • Quality reduced (often imperceptibly)
  • MP3, JPEG, MP4

Lossless

  • No data permanently lost
  • Original fully recoverable
  • Less size reduction
  • PNG, ZIP, FLAC

Run-Length Encoding (RLE)

A lossless method that replaces consecutive identical values with a count and the value.

Original: WWWWWBBBWWWWWWBB RLE: 5W 3B 6W 2B Saved: 16 tokens → 8 tokens
—

RLE works best on images with large uniform regions (logos, diagrams). It is poor for photographs where adjacent pixels differ constantly.

Chapter 1 — Summary

All key concepts at a glance.

Binary

Base-2. Each column is a power of 2. 8 bits = 1 byte = values 0–255.

Hexadecimal

Base-16 (0–9, A–F). 1 hex digit = 4 binary digits. Used for error codes, MAC addresses, colours, IPv6.

Binary Addition

Column by column with carries. Overflow if result exceeds available bits.

Logical Shifts

Left = ×2. Right = ÷2. Bits lost off either end cause errors.

Two's Complement

MSB is negative. Range −128 to +127 for 8 bits. Flip-and-add-1 to convert.

ASCII / Unicode

ASCII: 7-bit, 128 chars. Unicode: up to 4 bytes, all world languages.

Sound

Sampling rate (Hz) × bit depth determines quality and file size.

Images

File size = width × height × colour depth. More detail = bigger file.

Compression

Lossy = permanent data loss. Lossless = fully reversible. RLE is a lossless method.
