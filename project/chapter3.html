3.1 Computer Architecture

The CPU, Von Neumann architecture, registers, buses and performance factors.

The CPU

The Central Processing Unit executes all program instructions. Three main components:

  • Control Unit (CU) — manages and coordinates all activity
  • Arithmetic & Logic Unit (ALU) — performs calculations and logic operations
  • Registers and buses — tiny storage locations and the wires connecting them

Von Neumann Architecture

Developed in the 1940s — still the basis of all modern computers. Key concepts:

  • CPU can access memory directly
  • Memory stores both programs and data
  • Programs are sequences of instructions executed in order
CONTROL UNIT (CU)
PCProgram Counter
CIRCurrent Instr. Reg.
MARMemory Addr. Reg.
MDRMemory Data Reg.
ARITHMETIC & LOGIC UNIT (ALU)
ACC — Accumulator

Registers

Register Abbr. Purpose
Memory Address Register MAR Stores address of memory location being read or written
Memory Data Register MDR Stores data fetched from or about to be written to memory
Program Counter PC Stores address of the next instruction to be fetched
Current Instruction Register CIR Stores the instruction currently being decoded/executed
Accumulator ACC Holds results of ALU calculations temporarily

System Buses

Address Bus→ unidirectionalCPU → memory
Data Bus⇆ bidirectionalCPU ↔ memory ↔ I/O
Control Bus⇆ bidirectionalCU → components
—

A 16-bit address bus can address 2¹⁶ = 65 536 memory locations. A 32-bit bus can address over 4 billion. Wider buses = more addressable memory.

CPU Performance Factors

Factor Effect
Clock speed More cycles per second → faster processing. Modern CPUs ~3.5 GHz. Overclocking risks crashes and overheating.
Cache size More frequently used instructions stored near CPU → fewer slow RAM accesses.
Core count More tasks simultaneously. Doubling cores ≠ double performance (communication overhead).
Bus width Wider buses transfer more data per cycle.

Embedded Systems

An embedded system is hardware + software designed for one specific set of tasks. Found in cars, washing machines, traffic lights, microwaves.

Type What it contains
Microcontroller CPU + RAM + ROM + peripherals on one chip
Microprocessor CPU only — RAM/ROM added separately
System on Chip (SoC) CPU + memory + I/O + storage on one chip
—

A computer is NOT an embedded system. Computers are multi-functional; embedded systems perform one specific set of tasks.

Fetch–Decode–Execute Cycle

The repeating sequence the CPU follows to process every instruction.

FDE Cycle Steps

1. PC holds addressProgram Counter contains the address of the next instruction.
2. Address → MARPC contents copied to MAR via the address bus.
3. FetchInstruction at that memory address copied into MDR via data bus.
4. MDR → CIRContents of MDR copied into Current Instruction Register.
5. PC incrementedPC increased by 1, pointing to the next instruction.
6. DecodeCU decodes the instruction in CIR.
7. ExecuteControl signals sent via control bus to carry out the instruction.

Instruction Format

Part Purpose
Opcode The operation to perform (e.g. ADD, LDA, JMP)
Operand The data to act on, or a register/memory reference

The full set of operations a CPU understands is its instruction set. A common standard is X86 (Intel and AMD).

3.2 Input & Output Devices

How data enters and exits the computer.

Input Devices

🛒
Barcode Scanner
Red laser reads dark/light bars. Reflected light converted to digital data. Used in supermarkets, libraries.
📱
QR Code Scanner
Matrix of dark/light squares. Holds up to 4,296 characters. Read by smartphone cameras.
📷
Digital Camera
CCD converts light into electrical charges. ADC converts to 24-bit RGB digital images.
⌨️
Keyboard
Keypress completes a circuit. CPU looks up key and converts to ASCII value.
🎤
Microphone
Diaphragm vibrates; coil in magnetic field induces current. ADC converts to digital.
🖱️
Optical Mouse
LED bounces off surface; CMOS takes 1500 images/sec. DSP computes movement coordinates.
📄
2D Scanner
Light sweeps document; CCD converts reflected light to digital. OCR turns scanned text editable.
👆
Touch Screen
Capacitive (electrostatic field), infrared (IR beams), or resistive (pressure contact).

Output Devices

💧
Inkjet Printer
Thermal bubble or piezoelectric nozzles spray ink droplets. Good for photo printing.
⚡
Laser Printer
Laser charges drum; toner sticks to charged areas; fuser heat bonds toner to paper.
🧱
3D Printer
Builds objects layer by layer (~0.1 mm). Used in prosthetics, medical models, prototyping.
📺
LCD Screen
Liquid crystals controlled by voltage. Modern backlighting uses LED — brighter, thinner.
✨
OLED Screen
Each pixel emits its own light — no backlight. Thinner, flexible, lower power.
🔊
Speakers
DAC converts digital audio to current. Current through coil vibrates cone to produce sound.
⚙️
Actuators
Motors, solenoids, valves and relays that convert electrical signals into physical motion.
📽️
DLP Projector
Millions of micro-mirrors on a DMD chip tilt to direct light through colour filters.

Touch Screen Technologies

Type How it works Strengths Limitations
Capacitive Finger disrupts electrostatic field Excellent clarity, durable, multi-touch Bare fingers/stylus only
Infrared Finger breaks IR beam matrix Multi-touch, very durable Sensitive to water, accidental activation
Resistive Pressure makes two layers contact Works with gloves/stylus, dust/water resistant No multi-touch, scratches easily

3.2 Sensors & Control

Reading the physical world and acting on it.

Sensors

Sensors read analogue physical properties. An ADC converts the analogue signal to digital for the microprocessor. A DAC converts back to analogue to drive actuators.

Sensor Measures Example uses
Temperature Heat Central heating, greenhouses
Moisture Water in soil Irrigation, food processing
Light Brightness Auto street lights, car headlights
Passive IR Body heat radiation Burglar alarms, freezers
Pressure Force on sensor Weighing lorries, gas monitoring
Sound / Acoustic Sound waves Footstep detection, leak detection
Accelerometer Change in velocity Airbags, screen rotation
Proximity Nearby objects Phone screen off near ear
Gas CO₂, O₂ etc. Pollution monitoring
pH Acidity Soil analysis, chemical processes

Monitoring vs Control

Monitoring

Microprocessor watches only. Compares readings to stored values; triggers alarms if out of range. Does not change the environment.

e.g. patient vital signs, burglar alarms, pollution levels

Control

Microprocessor acts. Sends signals to actuators to change the environment. Output feeds back into the next sensor reading: a feedback loop.

e.g. central heating, ABS brakes, traffic lights, greenhouses

3.3 Data Storage

Primary, secondary and cloud storage.

RAM vs ROM

Feature RAM ROM
Volatile? Yes — data lost when power off No — retains data without power
Access Read and write Read only
Contains Running programs, open files, OS parts BIOS / bootstrap loader

DRAM vs SRAM

Feature DRAM (Dynamic) SRAM (Static)
Needs refresh? Yes (~every 15 ms) No
Speed ~60 ns ~25 ns
Cost Cheaper More expensive
Used in Main RAM CPU cache

Secondary Storage

Magnetic — HDD

  • Data on magnetic platters spinning ~7,000 rpm
  • Read-write heads read/write surfaces
  • Suffers from fragmentation and latency

Solid State — SSD, USB Flash, SD cards

  • No moving parts — NAND flash memory
  • Faster, quieter, more durable, lower power
  • Limited write cycles (very high in practice)

Optical — CD, DVD, Blu-ray

Disc Laser Wavelength Capacity
CD Red 780 nm 700 MB
DVD (single) Red 650 nm 4.7 GB
Blu-ray (single) Blue 405 nm 25 GB
Blu-ray (dual) Blue 405 nm 50 GB

Virtual Memory & Cloud

Virtual memory uses part of HDD/SSD as extra RAM when physical RAM fills up. Managed using paging.

—

Disk thrashing — when the system spends more time swapping data than processing. Fixed by adding RAM, running fewer programs, or using an SSD.

Cloud storage stores data on remote servers with redundancy. Access from anywhere — but requires reliable internet and raises security concerns.

3.4 Network Hardware

Components that enable devices to connect to networks.

NIC & MAC Address

A NIC (Network Interface Card) allows a device to connect to a network. Each NIC has a unique MAC address — a 48-bit identifier written as 6 hex pairs.

00 – 1C – B3 – 4F – 25 – FF ↑ manufacturer (3 pairs) ↑ device serial (3 pairs)

IP Addresses

Version Bits Format Example
IPv4 32 Four denary groups (0–255) 109.108.158.1
IPv6 128 Eight hex groups, colon-separated a8fb:7a88:fff0:…

Static IP

  • Permanently assigned by ISP
  • Doesn't change on reconnect
  • More expensive
  • Used by web/FTP servers

Dynamic IP (DHCP)

  • Assigned each session
  • Greater privacy
  • Cheaper
  • Can cause VoIP issues

MAC vs IP Address

MAC Address IP Address
Identifies physical device Identifies network address
48-bit, written in hex 32-bit (v4) or 128-bit (v6)
Set at manufacture Assigned by ISP / DHCP
Used within local network Used globally on the internet

Chapter 3 — Summary

All key hardware concepts at a glance.

CPU

CU + ALU + Registers + Buses. Von Neumann: programs and data share memory.

Registers

MAR, MDR, PC, CIR, ACC. Each has a specific role in the FDE cycle.

Buses

Address (uni) · Data (bi) · Control (bi). Width affects addressable memory and performance.

FDE Cycle

Fetch (PC→MAR→MDR→CIR) · Increment PC · Decode · Execute. Repeats endlessly.

Performance

Clock speed · cache · core count · bus width all affect CPU speed.

Embedded systems

One specific task. Microcontroller / microprocessor / SoC. Not a general computer.

Input devices

Barcode, QR, camera, keyboard, microphone, mouse, scanner, touch screen.

Output devices

Inkjet, laser, 3D printer, LCD, OLED, speakers, actuators, projectors.

Sensors & control

ADC converts analogue → digital. Monitoring = watch only. Control = feedback loop.

Storage

RAM (volatile) · ROM (non-volatile) · HDD · SSD · Optical · Cloud

Virtual memory

HDD/SSD as extra RAM. Paging. Thrashing = too much swapping.

Network hardware

NIC + MAC address. IP address (static / dynamic). IPv4 32-bit; IPv6 128-bit.

Glossary

Key terms from Chapter 3.

CPUCentral Processing Unit — executes program instructions. Contains the CU, ALU, and registers.
Von NeumannArchitecture where CPU accesses memory directly and memory stores both programs and data.
RegisterA tiny, extremely fast memory location inside the CPU used during the FDE cycle.
FDE CycleFetch–Decode–Execute — the repeating sequence a CPU follows to process every instruction.
CacheSmall, fast memory inside the CPU storing frequently used instructions/data to reduce RAM accesses.
Embedded systemHardware and software designed to perform one specific set of tasks.
ADCAnalogue-to-Digital Converter — converts continuous sensor signals into discrete digital values.
DACDigital-to-Analogue Converter — converts digital values back to analogue signals for output devices.
RAMRandom Access Memory — volatile, fast read/write memory for running programs.
ROMRead-Only Memory — non-volatile, permanent storage for the BIOS/bootstrap loader.
Virtual memoryA portion of HDD/SSD used as extra RAM when physical RAM is full, managed by paging.
MAC addressA unique 48-bit hardware identifier for a NIC, written as 6 hex pairs.
IP addressA numerical network label. IPv4 = 32-bit; IPv6 = 128-bit.
NICNetwork Interface Card — hardware that connects a device to a network.
