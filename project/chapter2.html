2.1 Data Packets & Packet Switching

How large messages are broken into small independent chunks and routed across networks.

Data Packets

When data travels long distances it is broken into data packets — typically ~64 KiB each. Each packet can take a different route and is reassembled at the destination.

HEADERIPs + sequence + size
PAYLOADactual data (~64 KiB)
TRAILERend marker + CRC

Header contains

  • IP address of sender and receiver
  • Sequence number — for correct reassembly
  • Packet size in bytes

Trailer contains

  • End-of-packet marker
  • CRC (Cyclic Redundancy Check) — error-checking value

Benefits

  • Each packet takes a different route
  • Only failed packet re-sent
  • No line permanently tied up
  • High transmission rates

Drawbacks

  • Packets may arrive out of order
  • Packets can be lost
  • Delay during reassembly
  • Issues with real-time streams

Packet Switching

Each packet travels independently through a network of routers. Each router reads the destination IP and decides the best next hop based on current queue lengths.

—

Hop count: To prevent packets looping forever, each packet has a hop counter decremented at every router. When it reaches zero the packet is discarded and re-requested.

2.1 USB

The universal standard for connecting devices to computers.

Universal Serial Bus

USB is a serial standard supporting half-duplex and full-duplex. A cable has 4 wires: 2 for power (+5V, ground), 2 for data.

Plug-in sequence

1

Computer detects voltage change on data wires.

2

Device recognised; appropriate device driver loaded.

3

If no driver found, user prompted to download one.

Benefits

  • Auto-detected; driver auto-loaded
  • Connector fits one way only
  • Industry standard
  • 1.5 Mbps – 5 Gbps supported
  • No external power needed
  • Built-in error detection
  • Backward compatible

Drawbacks

  • Max cable length: 5 metres
  • Very early USB not supported
  • USB V2 capped at 480 Mbps
—

USB-C: 24-pin symmetrical connector — fits either way. Supports 100 W power, 10 Gbps data, 4K video. Backward compatible via adaptor.

2.2 Error Detection

Five methods for detecting data corruption during transmission.

Why Error Detection?

Data can be corrupted, lost or gained due to electrical interference, skewing, or packet switching problems. Five methods:

—

Parity check · Checksum · Echo check · Check digits · ARQ

Parity Check

A parity bit is set to make the total count of 1-bits in a byte either even (even parity) or odd (odd parity). The receiver recounts and checks for a mismatch.

Limitation: if two bits flip, parity may still appear correct. Parity cannot detect even-numbered errors and cannot identify which bit is wrong.

—

Parity blocks add a parity byte at the end of a block. Checking both horizontally (per byte) and vertically (per column) pinpoints the exact corrupted bit.

Checksum

1

Sender calculates a checksum from the data block using an agreed algorithm.

2

Checksum is sent with the data.

3

Receiver recalculates the checksum from received data.

4

If checksums match → no error. If different → re-send.

Echo Check

1

Sender transmits data to receiver.

2

Receiver sends the data straight back.

3

Sender compares. Mismatch → error → re-send.

—

Flaw: if copies don't match you can't tell whether the error occurred on the way out or back. The simplest but least reliable method.

Check Digits

The check digit is the final digit of a code, calculated from all others. Used on barcodes, ISBNs and VINs to catch data entry errors.

Catches: incorrect digits, transposition (5037 ↔ 5307), omitted/extra digits.

ISBN-13

Sum of odd-position digits + (sum of even-position digits × 3). Divide by 10. If remainder = 0, check digit = 0; otherwise 10 − remainder.

Modulo-11

Assign weightings (8,7,6,5,4,3,2), multiply, sum, divide by 11. Check digit = 11 − remainder. (Remainder 10 → 'X')

—

The algorithm will be given in the exam. You need to understand how to apply it to calculate or verify a check digit.

ARQ — Automatic Repeat Request

1

Receiver checks data using an error-detection code (CRC).

2

No error → positive acknowledgement sent.

3

Error detected → negative acknowledgement; re-transmission requested.

4

Sender waits for a timeout period. No acknowledgement → auto re-send.

5

Repeats until positive acknowledgement or maximum retries reached.

2.3 Encryption

Symmetric and asymmetric encryption — protecting data in transit.

Purpose of Encryption

Encryption converts data into an unreadable form. It does not prevent interception — it prevents eavesdroppers from making sense of what they capture.

Encryption + key

Symmetric Encryption

One key encrypts and decrypts. Fast. Modern systems use 256-bit keys — 2²⁵⁶ ≈ 10⁷⁷ combinations.

—

Key distribution problem: the shared key must travel to the receiver and could be intercepted in transit.

Asymmetric Encryption

Uses two mathematically linked keys: a public key (shared openly) and a private key (kept secret). You cannot derive one from the other.

Example: Tom → Jane

1

Jane generates a key pair on her computer.

2

Jane shares her public key with Tom. Safe to intercept — it can only encrypt.

3

Tom encrypts using Jane's public key and sends ciphertext.

4

Jane decrypts with her private key. Only it can unlock messages encrypted with her public key.

The private key never travels. Even if the public key is intercepted, it cannot decrypt — only encrypt.

Chapter 2 — Summary

Key concepts at a glance.

Data packets

Header (IPs + sequence) · Payload (~64 KiB) · Trailer (end marker + CRC)

Packet switching

Routers choose paths; packets arrive out of order and are reassembled.

Direction modes

Simplex (one way) · Half-duplex (alternating) · Full-duplex (simultaneous)

Serial vs Parallel

Serial: 1 bit, 1 wire, long distance. Parallel: multi-bit, short distance, skewing risk.

USB

Serial · Half/full-duplex · 5 m max · Auto-detect · Backward compatible

Parity check

Counts 1-bits. Misses 2-bit errors. Parity blocks pinpoint exact bit.

Checksum

Calculated value sent with data. Receiver recalculates. Mismatch = error.

Echo check

Receiver returns data. Sender compares. Cannot tell where error occurred.

Check digits

Final digit derived from others. Catches ISBN, barcode scanning errors.

ARQ

Ack + timeout + re-send. Used in mobile networks.

Symmetric encryption

One key. Fast. Key distribution is the weakness.

Asymmetric encryption

Public key encrypts. Private key decrypts. Private key never travels.

Glossary

Key terms from Chapter 2.

Data packetA small unit of data transmitted over a network, containing a header, payload and trailer. Packets can take different routes to the destination.
CRCCyclic Redundancy Check — an error-checking value stored in a packet's trailer, derived from the payload bits.
RouterA network device that receives packets and decides the best next hop based on the destination IP address.
SimplexTransmission in one direction only.
Half-duplexTransmission in both directions, but not at the same time.
Full-duplexTransmission in both directions simultaneously.
SkewingIn parallel transmission, bits on separate wires arriving at slightly different times, causing data corruption.
Parity bitAn extra bit added to a byte to make the count of 1-bits even or odd, enabling simple error detection.
ChecksumA calculated value sent with a data block. Recalculated at the destination to verify integrity.
ARQAutomatic Repeat Request — error control using acknowledgements and timeouts to trigger re-transmission.
PlaintextThe original readable message before encryption.
CiphertextThe scrambled, unreadable output after encryption.
Symmetric encryptionEncryption using one key for both encrypting and decrypting.
Asymmetric encryptionEncryption using a matched public/private key pair. Public key encrypts; private key decrypts.
